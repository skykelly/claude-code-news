# 진행 상황 확인

`plan.md`와 `progress.md`를 읽어 현재 작업 상태를 아래 형식으로 요약한다.

## 출력 형식

```
📋 전체 태스크: N개
✅ 완료: N개  🔄 진행 중: N개  ⏳ 대기: N개  🚫 블로커: N개

완료된 태스크:
  ✅ TASK-00: 설명 (완료일)
  ✅ TASK-01: 설명 (완료일)

진행 중:
  🔄 TASK-02: 설명
     └ 현재 상황: ...

다음 실행 가능한 태스크:
  ⏳ TASK-03: 설명 (의존: TASK-02 완료 필요)

블로커:
  🚫 없음 또는 블로커 설명
```

## 추가 분석

- 전체 진행률 (완료 / 전체 × 100%)
- 의존 관계 기준으로 지금 당장 시작할 수 있는 태스크 목록
- 예상되는 다음 단계 제안
